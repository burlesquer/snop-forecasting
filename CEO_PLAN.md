## Vision

### 10x Check
"S&OP 데모"가 아니라 **"interactive S&OP decision support tool — 한국 K-뷰티 컨텍스트"**.

면접관이 5분 안에 본인이 단순히 forecasting을 "구현할 줄 안다"가 아니라 **"의사결정 도구로 작동시킬 줄 안다"**는 걸 경험하게 만드는 것. 단방향 보고서가 아니라 양방향 시뮬레이터.

### Platonic Ideal
면접관이 데모를 열고 5분 안에 경험하는 흐름:
1. **"오늘의 위험 SKU 3개"** → KPI 카드가 즉시 의사결정 정보 전달
2. **"내가 직접 발주해보기"** → SKU 선택 + 발주량 입력 → 향후 4주 KPI 시뮬레이션 변화
3. **"왜 이 모델이 X로 예측했는가"** → SHAP feature contribution 차트
4. **"프로모션 시나리오"** → A/B 비교 (이벤트 진행 vs 미진행)
5. **"신뢰구간 토글"** → 점추정 ± 90% CI 시각화

이 5개를 묶으면 단순 forecasting 데모 수준이 아니라 interactive decision support tool 수준.

## Engineering Preference Mapping
- **DRY**: ML 코드는 `nixtla mlforecast` 활용 권장 (Naive/MA/LGBM 동일 API). 직접 구현 보일러플레이트 제거.
- **테스트**: 신뢰구간 quantile 정합성 (P10 ≤ P50 ≤ P90), What-if 시나리오 차이 sanity check, 시뮬레이터 결품확률 경계값 테스트.
- **관찰 가능성**: 데이터 파이프라인 각 단계 logging (subset row count, 학습 시간, 평가 메트릭).
- **명시성**: SHAP/quantile/scenario 등 모든 새 출력 필드는 `lib/types.ts`에 타입 정의.
